# Serverless Website Log Analytics Pipeline

This `README.md` outlines the setup and configuration of a complete serverless data analytics pipeline using Amazon S3, Amazon Athena, AWS IAM, and Power BI. 

---

## Phase 1: Data Architecture & Storage (Amazon S3)
This phase establishes a clean data lake directory layout where raw logs are separated from automated query outputs.

*   Prepare a local file named `website_logs.csv` containing 100 website traffic records.
*   Ensure the first row of your CSV contains exactly these headers: `timestamp`, `user_id`, `page`, `country`, `device`.
*   Log into the AWS Console, open Amazon S3, and create a bucket with a globally unique name, such as `website-log-data-2026`.
*   Inside this new bucket, create two separate folders to keep the pipeline clean: `raw-logs/` for raw incoming data files, and `athena-results/` for background CSV files generated by Athena.
*   Navigate inside the `raw-logs/` folder and upload the `website_logs.csv` file.

## Phase 2: Serverless Database & Schema Definition (Amazon Athena)
This phase configures AWS Glue and Athena to query the folder-isolated data using standard SQL and validates the data by running core queries.

*   Open Amazon Athena in the AWS Console, navigate to the Settings tab, and click Manage.
*   Set the Query result location to point directly to your results folder (e.g., `s3://website-log-data-2026/athena-results/`) and click Save.
*   In the query editor, spin up your logical database by running the command: `CREATE DATABASE website_analytics;`.
*   Select `website_analytics` from the Database dropdown menu and execute the `CREATE EXTERNAL TABLE website_logs` query.
*   Ensure the `LOCATION` string points strictly to the `raw-logs/` folder so it never reads the results folder recursively.
*   Run core analytical queries, such as selecting the total count of visits or grouping visits by country, to validate the schema and view aggregated metrics.

## Phase 3: Secure Cloud Connectivity (IAM & ODBC DSN)
This phase sets up a secure pipeline allowing a local machine to communicate with the AWS environment without using a root account.

*   Open the AWS IAM Console and create a dedicated user named `powerbi-athena-connector`.
*   Attach the `AmazonAthenaFullAccess` and `AmazonS3FullAccess` policies directly to this new user.
*   Navigate to the user's Security credentials tab, click Create access key, choose "Third-party service", and copy the new Access Key ID and Secret Access Key.
*   Download and run the official Simba Athena 64-bit ODBC Driver installer on your Windows machine.
*   Open ODBC Data Sources (64-bit), go to the System DSN tab, click Add, and select the Simba Athena ODBC Driver.
*   Configure the Data Source Name as `Athena_Website_Logs`, define your AWS Region, and set the S3 Output Location to your Athena results folder.
*   In Authentication Options, select "IAM Credentials", input your Access Key ID and Secret Access Key, and click Test to ensure a successful cloud connection.

## Phase 4: BI Dashboard Visualization (Power BI Desktop)
This phase imports the live cloud schema, transforms the data to remove CSV structural text, and generates an executive dashboard.

*   Open Power BI Desktop, click Get Data, select ODBC, choose `Athena_Website_Logs` from the dropdown, and click OK.
*   In the Navigator, expand `AwsDataCatalog`, select the `website_analytics` database, check the `website_logs` table, and click Transform Data.
*   Filter out the CSV header row by clicking the filter arrow on the `timestamp` column and unchecking the literal text value "timestamp".
*   Optimize the schema by right-clicking the `timestamp` column header and selecting Remove.
*   Click Close & Apply to load the transformed data into Power BI.
*   Build a **Total Visits Metric** by adding a Card visual and setting the `user_id` field to Count.
*   Build a **Geographic Traffic Distribution** visual by adding a Stacked Bar Chart with `country` on the Y-axis and `user_id` (Count) on the X-axis.
*   Build a **Platform Split** visual by adding a Pie Chart with `device` as the Legend and `user_id` (Count) as Values.
*   Build a **Page Engagement Performance** visual by adding a Stacked Column Chart with `page` on the X-axis and `user_id` (Count) on the Y-axis.
